    ## Tailwind CSS !important modifier

    <Services.ItemTags>
      <Services.ItemTag>tailwindcss</Services.ItemTag>
      <Services.ItemTag>css</Services.ItemTag>
    </Services.ItemTags>

    Make any utility important by adding a `!` character to the beginning:

    ```html
    <p class="font-bold !font-medium">
      This will be medium even though bold comes later in the CSS.
    </p>
    ```

    [Official Docs](https://tailwindcss.com/docs/configuration#important-modifier)

  </Services.Item>

    ## pre overflow: scroll inside flex

    <Services.ItemTags>
      <Services.ItemTag>css</Services.ItemTag>
      <Services.ItemTag>pre</Services.ItemTag>
      <Services.ItemTag>flex</Services.ItemTag>
    </Services.ItemTags>

    Pernah pake `overflow-x: scroll` pada `pre` element tapi ngga ngaruh, malah parent yang ada scrollnya?

    Mungkin parentnya didalem `display: flex`. Kalau iya, coba tambahin `min-width: 0` pada parentnya.

    [Credit](https://weArticles.west-wind.com/posts/2016/feb/15/flexbox-containers-pre-tags-and-managing-overflow)

    ## :not() pseudo-class

    <Services.ItemTags>
      <Services.ItemTag>css</Services.ItemTag>
      <Services.ItemTag>selector</Services.ItemTag>
    </Services.ItemTags>

    Pake `:not()` ini emang agak tricky, salah penggunaan hasilnya juga bisa ngga sesuai harapan.

    Contoh, styling inline `code` dimana bukan children dari `pre` ataupun `.code-block`. Penggunaannya:

    <Services.DnD>
      <Services.Dont>
        ```css {copy:false} {footer:false}
        :not(pre), :not(.code-block) {
          > code {
            background-color: 'red';
          }
        }
        ```
      </Services.Dont>
      <Services.Do>
        ```css {copy:false} {footer:false}
        :not(pre, .code-block) {
          > code {
            background-color: 'red';
          }
        }
        ```
      </Services.Do>
    </Services.DnD>

    ## Turborepo dependsOn

    <Services.ItemTags>
      <Services.ItemTag>turborepo</Services.ItemTag>
      <Services.ItemTag>monorepo</Services.ItemTag>
    </Services.ItemTags>

    ```json {copy:false} {footer:false}
    "build": {
      "dependsOn": ["^lint", "test"]
    },
    ```

    `dependsOn` dengan ^ menandakan task dari package lain, sedangkan yang tanpa ^ task dari package itu sendiri.

    Misal: app A pas mau build harus running tasks `lint` di semua package dependenciesnya, dan juga harus running task `test` app A itu sendiri.

  </Services.Item>
